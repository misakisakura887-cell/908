# Mirror-AI 产品需求文档 (PRD v3)

> 最后更新：2026-02-14
> 版本：v3.0 - 初期策略规划

---

## 一、产品定位

**人人可用的 AI 投资平台**

核心理念：**1 美金起投**，极低门槛让普通人也能参与专业级投资策略。

---

## 二、核心基础设施

### Hyperliquid (HL)
- **类型**：去中心化交易所 (DEX)
- **关键特性**：无需 KYC，**免税交易**
- **支持资产**：
  - 加密货币（BTC、ETH、Meme 币等）
  - **美股**（Traditional Finance 板块）
  - **黄金**等大宗商品
- **作用**：底层交易执行层

---

## 三、初期核心策略（三大主力）

### 策略一：黄金量化策略

**技术实现**
1. 使用**微软开源模型**进行量化开发
2. 对接数据源 API，让量化模型直接生成黄金买卖信号
3. 通过 Hyperliquid 平台直接进行黄金交易

**特点**
- 全自动化量化交易
- 实时信号生成
- 风险可控

---

### 策略二：BTC 量化策略

**技术实现**
1. 复用黄金策略的逻辑框架
2. 对接 Hyperliquid 提供的 BTC 价格走势等 API 数据
3. 将数据输入量化大模型进行训练与推理
4. 实现 BTC 的自动化交易

**特点**
- 与黄金策略共享底层架构
- 针对加密市场特性优化
- 高频交易能力

---

### 策略三：龙头策略（主观与量化结合）

**初期阶段：主观跟单**
- 以见崎个人小仓位为主
- 用户可跟单主观策略
- 交易标的：
  - 美股（AI 板块、存储板块）
  - BTC
  - 黄金
  - 其他主流资产

**后期演进：量化龙头策略**
- 开发量化模型
- 让模型自动跑龙头股
- 形成量化版本的龙头策略（策略四）

**特点**
- 灵活性高
- 覆盖多资产类别
- 主观 → 量化的平滑过渡

---

## 四、市场契机与核心优势

### 契机分析

1. **市场需求**
   - 当前处于加密周期的最后一年
   - BTC 存在一定风险，但大量 USDT 资金不愿离场
   - 投资者渴望吃到美股 AI 和存储板块的红利

2. **平台局限**
   - Hyperliquid 目前尚未对接港股
   - 重点放在美股红利股上

### 核心优势

**🎯 免税交易**
- 通过 Hyperliquid 走策略可实现**免税**
- 规避直接投资美股所需缴纳的高额税费
- 这是平台的核心竞争力

**💰 低门槛**
- 1 美金起投
- 普通人也能参与专业级策略

**🤖 AI 驱动**
- 量化模型自动化交易
- 策略持续优化

**🔄 策略演进**
- 量化模型训练完成前：跟单主观龙头策略（策略三）
- 量化模型跑通后：选择量化龙头策略（策略四）

## 五、用户使用流程

### 主页策略选择
用户可在主页根据需求自行选择：

| 策略 | 类型 | 适合人群 | 特点 |
|------|------|----------|------|
| 黄金量化 | 全自动量化 | 稳健型投资者 | 大宗商品对冲 |
| BTC 量化 | 全自动量化 | 加密货币玩家 | 高频交易 |
| 龙头主观 | 跟单 | 看好个人判断 | 灵活多资产 |
| 龙头量化 | 量化（开发中） | 全自动偏好 | 未来版本 |

### 投资流程
1. 连接钱包
2. 选择策略
3. 输入金额（最低 1 美金）
4. 确认交易
5. 系统自动执行

## 六、商业模式

### 收费原则：**用户友好，无感付费**

| 收费项 | 金额 | 说明 |
|--------|------|------|
| License Fee | **0** | 不收取软件授权费 |
| 月费/年费 | **0** | 不收取订阅费 |
| 交易手续费 | 正常费率 + 小额加成 | 仅在交易时收取 |

### 示例
- 100 美金交易
- 原手续费：~3 美金
- Mirror-AI 收取：~5 美金（+2 美金用于 AI/Token 成本）
- 用户感知：几乎无感

## 七、技术架构

| 模块 | 说明 |
|------|------|
| 微软开源模型 | 黄金/BTC 量化策略核心 |
| Hyperliquid API | 交易执行、持仓管理、实时行情 |
| 数据源 API | 价格、走势、市场情绪数据 |
| AI 策略引擎 | 信号生成、风控、再平衡 |
| 钱包集成 | 多链多钱包支持 |
| 用户系统 | 账户、仓位、收益统计 |
| 安全防护 | 防 MEV、签名验证、权限隔离 |
